# Normalization Rules

Нормалізація виконується в окремому шарі між парсингом файлу та збереженням у БД.
Функція входу: `processRecords(records)` → повертає нормалізовані + валідовані записи.

---

## Загальні правила (для всіх полів)

| Правило         | Деталі                                                        |
| --------------- | ------------------------------------------------------------- |
| Trim            | Видалення пробілів на початку і в кінці                       |
| Collapse spaces | Кілька пробілів підряд → один                                 |
| NULL-значення   | `""`, `"-"`, `"N/A"`, `"невідомо"`, `"відсутній"` → `NULL`    |
| Не вигадувати   | Якщо значення відсутнє — залишати `NULL`, не підставляти фейк |
| Не перекладати  | Зберігати мову оригіналу                                      |

---

## Земля (Land)

### `cadastral_number` — кадастровий номер

- Видалити всі пробіли
- Залишити лише цифри та `:`
- Очікуваний формат: `XXXXXXXXXX:XX:XXX:XXXX`

```
"3222 987300:01:001:0023" → "3222987300:01:001:0023"
"3222987300-01-001-0023" → NULL  (невірний формат)
```

### `owner_name` — власник / землекористувач

- Trim + collapse spaces
- Title Case (перша літера кожного слова — велика)

```
"іванов іван іванович" → "Іванов Іван Іванович"
```

### `right_type` — тип права

Маппінг (case-insensitive):

| Вхідне значення                    | Результат   |
| ---------------------------------- | ----------- |
| `"власність"`, `"право власності"` | `OWNERSHIP` |
| `"оренда"`                         | `LEASE`     |
| будь-яке інше                      | `USAGE`     |

### `right_subtype` — підтип права

| Вхідне значення          | Результат                      |
| ------------------------ | ------------------------------ |
| `"заповіт"`, `"Заповіт"` | `за заповітом`                 |
| інші відомі варіанти     | нормалізований рядок           |
| невідоме                 | зберігається як є (після trim) |

### `registration_date` — дата реєстрації

Підтримувані формати вводу → результат `YYYY-MM-DD`:

```
"15.03.2021"     → 2021-03-15
"15/03/2021"     → 2021-03-15
"2021-03-15"     → 2021-03-15
Date object      → ISO date
невалідна дата   → NULL
```

### `area` (`square`) — площа, га

- Замінити `,` на `.`
- Видалити одиниці вимірювання (`га`, `m²` тощо)
- Привести до `float`
- Якщо парсинг не вдався → `NULL`

```
"12,5 га" → 12.5
"8.0"     → 8.0
"abc"     → NULL
```

## Нерухомість (Realty)

### `object_type` — тип об'єкта

Маппінг (lowercase, trim):

| Вхідне значення                                     | Результат              |
| --------------------------------------------------- | ---------------------- |
| `"квартира"`, `"житлова квартира"`                  | `Квартира`             |
| `"будинок"`, `"житловий будинок"`                   | `Будинок`              |
| `"нежитлове приміщення"`, `"комерційне приміщення"` | `Нежитлове приміщення` |
| `"земельна ділянка"`                                | `Земельна ділянка`     |
| невідоме                                            | `Інше`                 |

### `object_name` — назва об'єкта

- Trim + collapse spaces
- Видалити повторювані слова підряд
- Зберігати мову і регістр оригіналу (не перекладати, не змінювати case)

```
"Квартира   квартира № 5" → "Квартира № 5"
```

### `address` — адреса

Спроба розбити на структуровані поля:

| Поле               | Приклад                |
| ------------------ | ---------------------- |
| `region`           | `Київська область`     |
| `district`         | `Бориспільський район` |
| `city`             | `м. Бориспіль`         |
| `street`           | `вулиця Центральна`    |
| `building_number`  | `12А`                  |
| `apartment_number` | `47`                   |

Нормалізація абревіатур:

| Скорочення | Повна форма |
| ---------- | ----------- |
| `вул.`     | `вулиця`    |
| `просп.`   | `проспект`  |
| `пров.`    | `провулок`  |
| `бульв.`   | `бульвар`   |

- Видалити зайві коми та подвійні пробіли
- Якщо повний розбір неможливий → зберігати очищену повну адресу в `address_full`

### `area` (`total_area`) — загальна площа

- Замінити `,` на `.`
- Видалити одиниці (`м2`, `кв.м`, `кв.м.`, `m²`)
- Привести до `float`
- Якщо парсинг не вдався → `NULL`

```
"120,5 кв.м"  → 120.5
"85 м2"       → 85.0
"--"           → NULL
```

### `ownership_share` — частка власності

Підтримувані формати → `float` від `0.0` до `1.0`:

```
"1/2"   → 0.5
"1/4"   → 0.25
"100%"  → 1.0
"75%"   → 0.75
"0.5"   → 0.5
```

- Дріб `a/b` → `a / b`
- Відсоток `x%` → `x / 100`
- Якщо результат поза `[0, 1]` або парсинг не вдався → `NULL`

### `registration_number` — реєстраційний номер

- Trim + видалити пробіли
- Залишити лише літери та цифри
- Привести до UPPERCASE

```
"№ 123 abc-45" → "123ABC45"
```

### `object_id` — ідентифікатор об'єкта

- Trim + видалити пробіли
- Зберігати як рядок без змін (стабільний ідентифікатор)
- Не перетворювати тип

### `floor` — поверх

- Видалити текст (`"поверх"`, `"эт."`, `"пов."`)
- Витягти ціле число
- Якщо діапазон → взяти перше значення
- Якщо парсинг не вдався → `NULL`

```
"3 поверх"  → 3
"2-4"       → 2
"мансарда"  → NULL
```

### `total_floors` — кількість поверхів

- Видалити текст
- Ціле число
- Якщо парсинг не вдався → `NULL`

### `construction_year` — рік побудови

- Витягти 4-значний рік
- Дійсний діапазон: `1800 – поточний рік`
- Поза діапазоном або невалідне → `NULL`

```
"1978 р."  → 1978
"рік 2003" → 2003
"1650"     → NULL  (поза діапазоном)
```

### `material` — матеріал стін

Маппінг (case-insensitive):

| Вхідне значення           | Результат  |
| ------------------------- | ---------- |
| `"цегла"`, `"цегляний"`   | `BRICK`    |
| `"панель"`, `"панельний"` | `PANEL`    |
| `"бетон"`, `"монолітний"` | `CONCRETE` |
| невідоме                  | `OTHER`    |

## Валідація

Після нормалізації кожен запис отримує два нових поля:

| Поле                | Тип                  | Опис                 |
| ------------------- | -------------------- | -------------------- |
| `validation_status` | `VALID` \| `INVALID` | Підсумковий статус   |
| `validation_errors` | `string[]`           | Список кодів помилок |

### Коди помилок (Land)

| Код                        | Умова                                |
| -------------------------- | ------------------------------------ |
| `MISSING_CADASTRAL`        | `cadastral_number` відсутній         |
| `INVALID_CADASTRAL_FORMAT` | Присутній, але не відповідає формату |
| `MISSING_RIGHT_TYPE`       | `right_type` відсутній               |
| `MISSING_OWNER`            | `owner_name` відсутній               |
| `MISSING_DATE`             | `registration_date` відсутня         |
| `MISSING_DOCUMENT`         | `document_number` відсутній          |

### Правила маршрутизації

| Статус                | Куди потрапляє                                                |
| --------------------- | ------------------------------------------------------------- |
| `VALID` + новий запис | `registry` (основна таблиця)                                  |
| `VALID` + дублікат    | `crm` (для перевірки)                                         |
| `INVALID`             | `crm` з заповненими `validation_status` і `validation_errors` |

Записи **ніколи не видаляються** — `INVALID` зберігаються в CRM для ручного виправлення.

---

## Дедублікація

Перед збереженням видаляються точні дублікати в межах одного завантаження.

**Ключ дедублікації (Land):**

```
(cadastral_number, owner_name, right_type, registration_date)
```

Перший запис з групи зберігається, решта — відкидаються.
